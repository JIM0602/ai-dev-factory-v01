## 页面清单总览

本小程序采用"双端合一"架构（H6 假设），用户端和商家端共用一个微信小程序，通过角色识别展示不同界面。Web 后台为独立系统。

| 端 | 页面数量 |
|----|:---:|
| **用户端小程序**（顾客） | 5 |
| **商家端小程序**（移动管理） | 8 |
| **商家 Web 后台**（桌面管理） | 8 |
| **合计** | **21** |

## 一、用户端小程序（顾客端）

> 顾客打开小程序后默认进入用户端界面。如果同一个微信账号同时也是商家，需要提供角色切换入口。

### P01 - 门店主页（首页）

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/store/index` |
| **页面名称** | 门店主页 |
| **所属端** | 用户端小程序 |
| **主要功能** | 向顾客展示门店信息，提供预约入口 |

**页面元素**：
- 门店照片轮播（顶部 Banner，3-9 张图自动/手动轮播）
- 门店名称（大字突出展示）
- 门店地址 + 导航按钮（点击唤起微信地图导航）
- 门牌指引文字（如"3 楼电梯右转 302"）
- 营业时间 + 营业状态（"营业中"/"休息中"）
- 今日剩余可预约数（"今日还可预约 X 个时段"）
- 门店介绍文案
- **底部固定按钮**："立即预约"（核心 CTA）

**交互要点**：
- 下拉刷新页面。
- 点击地址区域唤起微信地图。
- 点击"立即预约"跳转到预约页 P02。

---

### P02 - 预约页面

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/reservation/create` |
| **页面名称** | 发起预约 |
| **所属端** | 用户端小程序 |
| **主要功能** | 顾客选择日期、时段、人数后提交预约 |

**页面元素**：
- 步骤指示器（可选，V1 可不用）
- 日期选择器：横向滚动日期条，默认选中"今天"，灰色显示休息日和过期日期
- 时段列表：纵向列表，每行显示时段时间（如 10:00-11:00）+ 剩余容量（如"余 3 位"）。无剩余容量的时段置灰不可选
- 人数选择器：+/- 按钮，默认 1 人，范围 1-10
- 手机号输入框：自动获取微信手机号或手动输入
- 备注输入框（选填）：文字备注，最长 100 字
- **底部按钮**："确认预约"

**交互要点**：
- 选择日期后，时段列表动态刷新（展示该日期的容量情况）。
- 时段置灰时点击无反应，并提示"该时段已满"。
- 点击"确认预约"后：
  - 容量充足 -> 跳转到预约结果页 P03（成功）。
  - 容量不足 -> 弹窗提示"该时段已约满，请重新选择"。

### P03 - 预约结果页

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/reservation/result` |
| **页面名称** | 预约结果 |
| **所属端** | 用户端小程序 |
| **主要功能** | 展示预约提交后的结果 |

**页面元素**：

- 结果文案："预约已提交，等待商家确认" / "预约成功！按时到店即可"（取决于商家是否开启"预约需确认"）
- 预约信息摘要：日期、时段、门店地址
- 提示文案：如"商家确认后您将收到通知" / "请在预约时间前到达"
- **底部按钮**："查看我的预约"（跳转到 P04 列表页）、"返回首页"（跳转到 P01）

**交互要点**：
- 从预约页 P02 提交后跳转至此。
- 不可直接通过 URL 访问（需携带提交结果参数）。

### P04 - 我的预约列表

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/reservation/my-list` |
| **页面名称** | 我的预约 |
| **所属端** | 用户端小程序 |
| **主要功能** | 查看所有预约记录，可筛选状态，可取消预约 |

**页面元素**：
- 顶部状态筛选 Tab：全部 / 待确认 / 已确认 / 已完成 / 已取消
- 预约卡片列表（按时间倒序），每张卡片显示：
  - 预约日期（如"5 月 11 日 周六"）
  - 时段（如"14:00 - 15:00"）
  - 人数（如"2 人"）
  - 状态标签（待确认=黄色、已确认=绿色、已拒绝=红色、已取消=灰色、已完成=蓝色）
- 空态：无预约时显示占位图 + "还没有预约记录，去预约吧" + 跳转按钮

**交互要点**：
- 点击状态 Tab 筛选列表。
- 点击卡片进入预约详情 P05。
- 下拉刷新列表。
- 符合条件的预约（已确认、未到截止时间）卡片上显示"取消预约"按钮。

### P05 - 预约详情页

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/reservation/detail` |
| **页面名称** | 预约详情 |
| **所属端** | 用户端小程序 |
| **主要功能** | 查看单条预约的详细信息，可取消预约 |

**页面元素**：
- 预约状态（大号状态标签 + 说明文案）
- 预约信息区：
  - 门店名称
  - 预约日期、时段、人数
  - 预约提交时间
  - 手机号
  - 备注
- 状态时间线（可选，V1 可简化）：
  - 提交预约 -> 商家确认 -> 到店登记 -> 计时开始 -> 计时结束
- 操作区：
  - "取消预约"按钮（仅"已确认"或"待确认"状态、且在截止时间前显示）

  - "导航到店"按钮（唤起微信地图）

**交互要点**：
- 点击"取消预约"弹出二次确认："确认取消该预约？取消后如需重新预约需再次选择时段。"
- 确认取消后跳转回 P04 列表，该条状态变为"已取消"。

| 属性 | 内容 |
|------|------|

| **所属端** | 商家端小程序 |
| **主要功能** | 查看和筛选所有预约，快速处理预约 |

**页面元素**：
- 搜索栏：支持按手机号/姓名搜索
- 日期筛选：今天 / 明天 / 选择日期
- 状态筛选：全部 / 待确认 / 已确认 / 计时中 / 已完成 / 已取消
- 预约卡片列表，每张卡片显示：
  - 顾客姓名 + 手机号（脱敏）

### B03 - 预约详情（商家）

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/merchant/reservation-detail` |
| **页面名称** | 预约详情（商家视角） |
| **所属端** | 商家端小程序 |
| **主要功能** | 查看单条预约完整信息，进行确认/拒绝/到店登记等操作 |

**页面元素**：

### B04 - 手动添加预约

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/merchant/add-reservation` |
| **页面名称** | 添加预约 |
| **所属端** | 商家端小程序 |
| **主要功能** | 商家为通过电话/微信联系预约的顾客手动录入预约 |

**页面元素**：
- 表单区：
  - 顾客姓名（必填，文本输入）
  - 手机号（必填，数字输入，11 位校验）
  - 预约日期（日历选择器，不能选过去日期和休息日）
  - 预约时段（下拉选择或列表选择，显示剩余容量）
  - 人数（默认 1，+/- 调整）
  - 备注（选填）
- 容量提示："该时段还可预约 X 个桌位"
- **底部按钮**："提交预约"

**交互要点**：
- 选择日期后时段列表动态刷新。
- 时段容量不足时置灰提示。
- 提交时校验必填项，通过后直接生成"已确认"预约。
- 提交成功后跳转到预约列表 B02，并 Toast 提示"预约添加成功"。

### B05 - 计时看板

| 属性 | 内容 |
|------|------|
| **路径** | `/pages/merchant/timer-dashboard` |
| **页面名称** | 计时看板 |
| **所属端** | 商家端小程序 |
| **主要功能** | 查看所有正在计时的顾客，进行加时/结束操作 |

**页面元素**：
- 顶部统计条：正在计时 X 组 | 空闲桌位 Y 个
- 计时顾客卡片列表（按剩余时间升序排列），每张卡片显示：
  - 顾客姓名（大字）
  - 桌位号（如"3 号桌"）
  - 预约时段（如"14:00-15:00"）
  - 已用时长（如"已过 32 分钟"）
  - 剩余时长（大号倒计时，如"剩余 28:15"）
  - 剩余 < 15 分钟：卡片边框变橙色
  - 剩余 < 5 分钟：卡片边框变红色，文字闪烁

  - "加时"按钮（蓝色）

**交互要点**：
- 倒计时每 10 秒自动刷新。
- 点击"加时"弹出时长选择器：+30 分钟 / +1 小时 / +2 小时 / 自定义。
- 点击"结束计时"弹出二次确认："确认 [顾客名] 离店并结束计时？"
- 操作后卡片从列表移除（结束）或倒计时更新（加时）。
