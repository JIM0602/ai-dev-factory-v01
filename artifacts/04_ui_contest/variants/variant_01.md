# UI 方案变体 1：温暖手作 Warm Handcraft

## 1. 方案名称与定位

| 属性 | 内容 |
|------|------|
| 方案名称 | 温暖手作 Warm Handcraft |
| 核心关键词 | 暖色调、圆润、亲切、针织感、手工温度 |
| 适合门店类型 | 文艺清新拼豆店、亲子温馨型门店、主打治愈体验的小型手作工作室 |
| 适合用户群 | 18-30 岁女性为主，偏好温暖治愈风格，注重门店铺的亲和力和舒适感 |
| 设计理念 | 模拟手作工作室的温暖氛围——像走进一个毛线、布料、拼豆交织的温馨空间。大量圆角、柔和阴影、暖色渐变，让用户在使用过程中感受到"手工的温度"。配色参考 Pantone 暖色系，避免冷冰冰的科技感。 |

---

## 2. 视觉风格关键词

温暖、圆润、毛毡质感、奶油色调、手写感、针织纹理元素

## 6. 交互细节

### 6.1 页面转场
- 小程序使用微信默认右滑进入/左滑返回的 slide 转场
- 弹窗从底部滑入（transform translateY），带 ease-out 缓动 300ms
- 弹窗背景遮罩 fade-in 200ms

### 6.2 关键反馈动画
- **预约提交成功**：按钮变为成功状态（绿色），缩小 5% 再弹回（spring 弹性动画），然后跳转结果页
- **确认预约**：卡片向左滑出 50px 同时淡出，列表其他项上移填补，总时长 300ms
- **计时到期提醒**（剩余<5分钟）：数字颜色脉冲闪烁（opacity: 1 → 0.5 → 1），周期 1.5s
- **到店登记成功**：全屏短暂绿色遮罩（opacity 0.2）闪一下 + Toast "已开始计时"
- **结束计时**：卡片缩小+淡出 250ms，下一个卡片上移

### 6.3 手势操作
- 预约卡片支持左滑显示操作按钮（"拒绝" "确认"），使用微信小程序原生 movable-view 或自定义手势
- 下拉刷新：页面整体下拉时显示暖色调加载指示器
- 计时卡片支持长按 500ms 弹出操作菜单（加时/结束）

## 7. 优点

1. **温暖治愈，贴合拼豆手作品类调性**。暖色调和圆润设计营造出"手工温度"感，目标女性用户情感认同度高。
2. **状态可视化优秀**。彩色状态条+边框变色方案，商家一眼识别预约和计时状态，效率高。
3. **层级清晰**。不同深浅的暖色背景（而不是纯白）构建了自然的视觉层次。
4. **计时看板视觉冲击强**。特大倒计时数字+颜色变化，商家不易错过即将到期的顾客。
5. **三端一致性强**。小程序和 Web 共享同一套 Design Token，品牌认知统一。

## 8. 缺点

1. **暖色系在强光下对比度可能不足**。户外阳光下使用小程序时，浅暖色底可能反光。
2. **Web 后台效率略受影响**。大量暖色和圆角在密集数据表格中不如冷峻的极简风高效。
3. **颜色识别的可访问性问题**。红绿色盲用户可能难以区分"已确认"和"待确认"状态标签（可加图标辅助）。
4. **毛毡/手作质感需要高还原度**。如果前端实现不到位，可能显得"土"而不是"温暖"。

## 9. 实现难度

**中等偏低**。主要是颜色系统和圆角规则的变化，不涉及复杂动效或非标准组件。uni-app 和 Element Plus 均原生支持。

## 10. 可复用性

**高**。Design Token 体系完整（色值、圆角、间距都有明确规范），可作为同类"温暖风格"零售/手作类小程序的基准模板。
