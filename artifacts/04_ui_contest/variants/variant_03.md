# UI 方案变体 3：极简轻量 Minimal Light

## 1. 方案名称与定位

| 属性 | 内容 |
|------|------|
| 方案名称 | 极简轻量 Minimal Light |
| 核心关键词 | 克制留白、高对比度、无印良品风、效率优先、轻线条 |
| 适合门店类型 | 日式极简风拼豆店、高端手工工作室、追求效率和专业感的门店、主打成人手工课程的门店 |
| 适合用户群 | 25-40 岁注重品质和效率的用户，喜欢清爽、无干扰的交互体验，对"少即是多"有认同感 |
| 设计理念 | "消除一切不必要的元素"。用大量留白、清晰的层级、克制的色彩让用户的注意力完全聚焦在内容和操作上。借鉴日本无印良品式的"空"美学——设计的存在是为了消失，让用户感受不到设计的存在。预约三步完成，商家操作一目了然。 |

---

## 2. 视觉风格关键词

大量留白、极细线条、单色阶、灰度优先、功能性设计、瑞士平面风格、Helvetica 精神

---

## 3. 设计系统 Design Tokens

### 3.1 色彩系统

核心原则：只用 1 个主色 + 灰度系统。色彩仅用于关键状态传达。

| 角色 | 色值 | 用途 |
|------|------|------|
| **主色 Primary** | `#2D2D2D` | 主要文字、主按钮背景、图标、选中态 |
| **主色浅** | `#5C5C5C` | 次要文字、次级按钮 |
| **强调色 Accent** | `#3B82F6` | 链接、唯一彩色点缀（用于最关键的状态区分） |
| **背景色 Page BG** | `#FAFAFA` | 全局页面底色（极浅灰） |
| **卡片/容器 BG** | `#FFFFFF` | 卡片、弹窗、列表项 |
| **分割线 Border** | `#EBEBEB` | 极轻分割线（几乎是白色） |
| **输入框边框** | `#D4D4D4` | 输入框常态边框 |
| **输入框聚焦** | `#2D2D2D` | 输入框聚焦边框 |
| **主文字 Text Primary** | `#1A1A1A` | 标题、重点正文 |
| **辅文字 Text Secondary** | `#737373` | 辅助说明 |
| **更轻文字** | `#A3A3A3` | 占位符、禁用态、提示文字 |
| **悬停/选中背景** | `#F5F5F5` | 列表项悬停、选中背景 |

**状态色（仅在需要传达状态时使用，其余一律灰度）**

| 状态 | 色值 | 说明 |
|------|------|------|
| 待确认 | `#2D2D2D` 黑点空心圆 | 空心圆圈表示等待中 |
| 已确认 | `#2D2D2D` 黑实心圆 | 实心圆表示已确认 |
| 已拒绝 | `#D4D4D4` 灰实心圆 + 删除线 | 灰调表示否定 |
| 已完成 | `#3B82F6` 蓝点（唯一彩点） | 蓝色标记已完成 |
| 已取消 | `#D4D4D4` 灰实心圆 + 删除线 | 与已拒绝同视觉处理 |
| 计时预警 | 数字右侧出现细进度条，颜色从黑渐变为 `#3B82F6` | 用进度而非颜色变化传达紧迫感 |
| 计时紧急 | 数字加粗 + 下划线 `#3B82F6` | 极简强调方式 |

### 3.2 字体层级

核心原则：整个应用只用 2 种字重（Regular 400 和 Medium 500），通过字号和间距区分层级，不用 Bold。

小程序端：

| 层级 | 字重 | 字号 | 字间距 | 用途 |
|------|------|------|--------|------|
| H1 | Medium (500) | 36rpx | 4rpx | 页面主标题 |
| H2 | Medium (500) | 32rpx | 2rpx | 区块标题 |
| H3 | Medium (500) | 28rpx | — | 卡片标题 |
| Body | Regular (400) | 28rpx | — | 正文 |
| Caption | Regular (400) | 24rpx | — | 辅助文字 |
| Small | Regular (400) | 22rpx | — | 标签、角标 |

Web 后台：

| 层级 | 字重 | 字号 |
|------|------|------|
| H1 | Medium (500) | 22px |
| H2 | Medium (500) | 18px |
| H3 | Medium (500) | 15px |
| Body | Regular (400) | 14px |
| Caption | Regular (400) | 12px |

### 3.3 间距系统

极简设计的灵魂在间距。遵循 4px 基础网格（比常规 8px 更细）。

| Token | 值（小程序） | 值（Web） | 用途 |
|-------|------------|-----------|------|
| 2xs | 4rpx | 4px | 极小间距（图标与文字） |
| xs | 8rpx | 8px | 紧密间距 |
| sm | 16rpx | 16px | 常规间距 |
| md | 24rpx | 24px | 卡片内边距 |
| lg | 40rpx | 40px | 页面水平边距（留白核心） |
| xl | 64rpx | 64px | 大区块间隔 |
| xxl | 96rpx | 96px | 页面顶部/底部留白 |

**关键留白规则：**
- 页面左右留白统一 40rpx（不是常见的 32rpx），刻意加宽营造呼吸感
- 卡片之间间距 24rpx
- 区块之间间距 48-64rpx
- 任何元素不贴边，最小间距 16rpx

### 3.4 圆角规则

极简风格圆角克制，接近直角但不完全直角。

| 元素 | 圆角值 |
|------|--------|
| 卡片 | 5rpx（近乎直角但柔和不刺眼） |
| 按钮 | 4rpx |
| 输入框 | 4rpx |
| 标签/徽章 | 2rpx（极微圆角） |
| 弹窗 | 8rpx |
| 图片容器 | 2rpx |

### 3.5 阴影与边框

极简设计**不使用阴影**。层次感通过微妙的边框和背景色差来实现。

| 元素 | 样式 |
|------|------|
| 卡片与背景的分界 | `0.5rpx solid #EBEBEB`（极细边框，几乎看不见但确实存在） |
| 悬浮态/选中态 | 背景色变化 `#FAFAFA` → `#F5F5F5`，无阴影 |
| 弹窗与背景的分界 | 弹窗背景 `#FFFFFF` vs 遮罩 `rgba(0,0,0,0.5)` |
| 底部操作栏 | 顶部 `0.5rpx solid #EBEBEB` |

### 3.6 线条系统

| 元素 | 规格 |
|------|------|
| 分割线 | `0.5rpx solid #EBEBEB`，贯通全宽 |
| 激活指示器 | `4rpx 高 × 32rpx 宽`，`#2D2D2D` 背景，圆角 2rpx（Tab 下划线） |
| 进度条 | `4rpx 高`，圆角 2rpx，背景 `#EBEBEB`，填充 `#2D2D2D` |
| 图标 | 线性图标（stroke 风格），`1.5rpx` 描边宽度 |

## 5. 组件设计

### 5.1 按钮

| 类型 | 样式规范 |
|------|----------|
| **主按钮 Primary** | 背景 `#2D2D2D`，文字白色，高度 88rpx，圆角 4rpx，字重 Medium。按下态：背景 `#1A1A1A`。宽度撑满容器或 240rpx 最小宽度 |
| **次要按钮 Secondary** | 背景透明，边框 `1rpx solid #2D2D2D`，文字 `#2D2D2D`，同高度 |
| **文字按钮 Text** | 无背景无边框，文字 `#2D2D2D`，按下态加下划线 |
| **危险操作** | 文字红色 `#DC2626`，无背景无边框（用红色文字本身传达危险感，而非红框按钮） |
| **确认/拒绝按钮（卡片内）** | 小型 56rpx 高，无背景，仅文字。确认=黑字，拒绝=灰字 `#A3A3A3` |

**按钮文案原则：** 只用最少的词。如"预约"而非"立即预约"，"确认"而非"确认预约"。

### 5.2 卡片

仅使用一种卡片样式，通过分割线区分内容：

```
┌──────────────────────────────┐
│                              │  ← 白色背景
│  内容                         │     内边距 md(24rpx)
│                              │     无阴影
│                              │     0.5rpx 细边框 #EBEBEB
└──────────────────────────────┘
```

无彩色顶条、无阴影、无圆角装饰。纯粹功能性的内容容器。

### 5.3 状态标识

极简方案——只用几何图形标识状态，不需要颜色：

| 状态 | 标识方式 |
|------|----------|
| 待确认 | ○ 空心圆 (border: 1.5rpx solid #2D2D2D) |
| 已确认 | ● 实心圆 (background: #2D2D2D) |
| 计时中 | ◉ 实心圆 + 外环 (background: #2D2D2D, border: 1.5rpx #2D2D2D) |
| 已完成 | ● 实心蓝点 (background: #3B82F6，唯一用色之处) |
| 已拒绝 | ○ 空心圆 + 斜线 (text-decoration: line-through) |
| 已取消 | ○ 空心灰圆 (border-color: #D4D4D4) |

### 5.4 进度条（计时看板）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  4rpx 高度
```

- 高度 4rpx，圆角 2rpx
- 背景 `#EBEBEB`
- 填充 `#2D2D2D`（始终黑色）
- 不因剩余时间少而变色——通过数字本身的变化表达紧迫感

### 5.5 空态

- 无插图，仅文字 + 引导按钮
- 文案极简："暂无预约" + 下方"去预约"文字链接
- 计时看板空态："暂无计时" + 下方一个细线

### 5.6 加载态

- 不使用骨架屏（骨架屏在极简风中显得多余）
- 使用 1 秒延迟的透明度过渡动画——内容从 opacity 0 → 1 fade-in
- 加载指示器：极细黑色圆环旋转（2rpx 宽线条）

### 5.7 错误态

- 表单错误：输入框下方红线闪烁一次（0.5rpx → 1rpx `#DC2626`），然后出现红色文字提示 12rpx
- 网络错误：页面顶部出现黑色细条（高度 2rpx），下方小字"网络连接异常"

## 6. 交互细节

### 6.1 页面转场

- 小程序使用微信默认转场，不额外定制
- 弹窗：fade-in 150ms（无位移，仅透明度变化），遮罩 fade 100ms

### 6.2 关键反馈动画

- **预约提交成功**：按钮文字变为"已提交"，同时在按钮右侧出现一个实心圆 ● 淡入（200ms）
- **确认预约**：卡片无动画，状态标识 ○ 变为 ● 平滑过渡（颜色填充动画 200ms）
- **计时到期提醒**：数字从 Regular 400 渐变为 Medium 500（字重过渡），同时下方进度条达到 90%+ 时进度条出现微妙的呼吸效果（透明度 1→0.6→1，周期 3s）
- **结束计时**：对应行淡出 300ms，下方内容平滑上移
- **所有操作反馈**：优先使用微小的状态改变（字重、透明度、位移 1-2px），而非大幅动效

### 6.3 手势操作

- 左滑操作：卡片仅向左滑动，露出下方操作区。无弹簧回弹——松手即定位（更干脆）
- 下拉刷新：微信原生风格
- 无长按操作（长按交互在极简设计中不直观）

## 7. 优点

1. **极致加载速度**。无阴影、无渐变、无色块，CSS 渲染开销极小，小程序性能最优。
2. **无障碍友好**。高对比度黑白设计天然支持视觉障碍用户，不依赖颜色传达信息。
3. **永不"过时"**。极简风格是时间的朋友，不会因潮流变化而显得老旧。
4. **商家端效率最高**。无视觉噪音，信息密度合理，商家可以最快速度找到目标预约并操作。
5. **开发实现最快**。设计系统极其简单（几乎只有黑白灰），CSS 变量最少，组件最少。
6. **品牌气质高级**。无印良品式的克制美学传递"品质感"，适合定位高端的门店。

## 8. 缺点

1. **缺乏情感温度**。极简可能显得"冷淡"，不太适合主打温馨亲子的拼豆店。
2. **门店辨识度弱**。没有色彩记忆点，用户难以在众多小程序中一眼认出。
3. **状态区分依赖认知**。空心圆/实心圆的状态编码需要用户学习，不如彩色标签一目了然。
4. **缺乏"趣味性"**。拼豆本身是充满色彩和创意的手作，极简设计与产品本质有一定张力。
5. **计时紧急状态不够醒目**。仅靠字重变化可能不够直观（V1 可考虑保留颜色作为唯一例外）。

## 9. 实现难度

**低**。设计系统极其简单，CSS 量少，组件变体少。开发可以在最短时间内完成高还原度的 UI。Element Plus 默认主题稍加调整即可适配。

## 10. 可复用性

**极高**。这个设计系统是纯粹的"功能容器"，几乎不包含任何品类特定元素。可以直接复用到任何工具型小程序或管理后台，只需替换一个强调色即可。
