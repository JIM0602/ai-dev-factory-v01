## 一、最终风格关键词

**温润、圆融、清晰、高效**

- **温润**：暖色调传递手工温度，不冰冷、不刺眼。
- **圆融**：圆角+柔阴影营造亲和力，避免尖锐直角。
- **清晰**：信息层级分明，状态一目了然，操作路径短。
- **高效**：商家端减少视觉噪音，Web 后台数据密度合理。

### 2.3 间距系统

基础单位：**8rpx（小程序）/ 8px（Web）**

| Token 名 | 小程序值 | Web 值 | 用途 |
|----------|---------|--------|------|
| `--space-xs` | 8rpx | 8px | 紧凑内边距、图标与文字间距 |
| `--space-sm` | 16rpx | 16px | 普通内边距、标签间距、按钮内边距 |
| `--space-md` | 24rpx | 24px | 卡片内边距、列表项间距 |
| `--space-lg` | 36rpx | 32px | 页面左右页边距（小程序加宽 4rpx 增强呼吸感） |
| `--space-xl` | 48rpx | 48px | 大区块间隔、底部操作区上方间距 |
| `--space-xxl` | 64rpx | 64px | 页面顶部/底部安全距离 |

**页面安全区**：
- 小程序底部安全区（含 home indicator）：`env(safe-area-inset-bottom)` 或最小 68rpx
- 小程序顶部状态栏 + 导航栏：系统自动处理，内容区从导航栏下方开始
- Web 后台内容区最小宽度：1024px

## 三、页面布局规范

### 3.1 通用页面结构（小程序端）

```
┌─────────────────────────────────┐
│         Status Bar（系统）       │
├─────────────────────────────────┤
│  导航栏（标题 + 返回/操作按钮）   │  ← 高度 88rpx（含状态栏），背景 #FFFFFF
├─────────────────────────────────┤
│                                 │
│                                 │
│         内容滚动区               │  ← flex: 1，overflow-y: auto
│     （页面左右边距 36rpx）       │     背景色 `--color-bg-page`
│                                 │
│                                 │
├─────────────────────────────────┤
│       底部固定操作栏（可选）      │  ← 上边阴影 L2，背景 #FFFFFF
│       底部安全区 68rpx          │
└─────────────────────────────────┘
```

**规则**：
- 导航栏标题居中对齐，字号 34rpx SemiBold，颜色 `#3D2C2E`
- 导航栏左侧返回按钮为向左箭头图标 + "返回"文字（可选），触控区域 ≥ 44rpx × 44rpx
- 页面内容区左右统一留白 `--space-lg`（36rpx）
- 内容区顶部与导航栏间距 `--space-md`（24rpx）
- 内容区底部与底部操作栏间距 `--space-xl`（48rpx）（无操作栏时用 `--space-xxl` 64rpx）
- 底部固定操作栏使用 `position: fixed; bottom: 0`，内部按钮距栏边 `--space-md`

**规则**：

- 表格卡片白色背景 + `1px solid #EBE5DC` 边框，无阴影
- 页面标题使用 H1 24px Bold，下方间距 24px

## 九、响应式规则

### 9.1 小程序端

- 微信小程序主要运行在手机竖屏环境，设计基准宽度为 **750rpx**
- 使用 rpx 单位自动适配不同屏幕宽度（1rpx = 屏幕宽度/750）
- 横屏场景不做特殊适配（V1 假设用户竖屏使用）

### 9.2 Web 后台

- **桌面端（≥1024px）**：标准布局。侧边栏 200px + 主内容区自适应。表格全列显示。
- **平板端（768px - 1023px）**：侧边栏折叠为图标模式（宽 64px），表格可横向滚动。V1 优先保证桌面端体验。
- **手机端（<768px）**：Web 后台不作为移动端主要使用场景（商家有商家端小程序）。如确实需要在手机浏览器打开，显示提示「请使用商家小程序操作」。

### 9.3 小程序 Web 后台共用原则

- 所有颜色 Token、间距 Token、字体 Token 在两端使用相同的命名
- CSS 变量在 `:root`（Web）和 `page`（小程序）中分别声明
- 组件逻辑可复用（Vue 3 组件），样式按端分别写在条件编译块中

## 十、前端开发必须遵守的清单

## 十一、禁止事项

以下是开发中**严格禁止**的做法：

| 编号 | 禁止事项 | 原因 |
|:----:|----------|------|
| 1 | **禁止硬编码颜色值** | 所有颜色必须使用 CSS 变量，确保后续可全局调整 |
| 2 | **禁止使用纯黑色 #000000** | 文字一律使用 `#3D2C2E`（深棕），纯黑在小程序中过于刺眼 |
| 3 | **禁止纯颜色区分状态** | 状态标签必须同时使用形状标识（◇●○■◉），色盲用户必须能区分 |
| 4 | **禁止触控区域 < 44rpx** | 微信小程序最小触控推荐值，小于此值用户难以点击 |
| 5 | **禁止使用 V02 的高饱和配色和硬阴影** | 与最终方案调性冲突。拼豆店不是潮玩店。 |
| 6 | **禁止使用 V04 的不对称圆角** | 四个角各自不同圆角在 uni-app 中实现成本过高，V1 不做 |
| 7 | **禁止在 Web 后台使用大面积暖色背景** | Web 后台页面底色统一用 `#FAFAF8`，卡片用白色+细边框 |
| 8 | **禁止引入外部动效库**（如 Lottie、GSAP） | V1 动效仅用 CSS transition/animation |
| 9 | **禁止使用图片实现空态插画** | 小程序包体积限制，用 CSS/SVG 内联 |
| 10 | **禁止弹窗中使用超过 2 个操作按钮** | 保持操作简洁，超过 2 个操作考虑使用 ActionSheet |
| 11 | **禁止在小程序中使用横向滚动表格** | 小程序不适合大表格，使用卡片列表代替 |
| 12 | **禁止按钮文案超过 8 个字** | 精简原则，超长文案使用 Text 按钮代替 |
